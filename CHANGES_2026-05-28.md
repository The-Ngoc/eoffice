# Ghi chú thay đổi dự án eOffice - Chiều 28/05/2026

Tài liệu này tóm tắt các phần đã chỉnh trong phiên làm việc chiều nay để tiện review, demo và push lên GitHub.

## 1. Dashboard nhân viên / chuyên viên

- Hoàn thiện màn hình chi tiết task cho chuyên viên theo hướng đọc dữ liệu thật từ backend.
- Tách rõ các nhóm file:
  - Tệp văn bản gốc.
  - Tệp đính kèm task.
  - File chuyên viên đã nộp.
  - File chuẩn bị nộp lại khi task bị từ chối.
- Thêm thông báo sau khi cập nhật tiến độ hoặc submit task.
- Bỏ phần “Bối cảnh chỉ đạo” không cần thiết trong vai trò nhân viên.
- Sửa các lỗi font tiếng Việt bị mojibake ở nhiều màn hình.

## 2. Render file đính kèm

- Chuẩn hóa cách hiển thị file ở các màn hình:
  - Văn thư.
  - Lãnh đạo.
  - Trưởng phòng.
  - Chuyên viên.
- Thêm helper `toDisplayFiles` để UI đọc được cả hai kiểu dữ liệu file:
  - `file_name/file_url`.
  - `nameFile/url`.
- Sửa lỗi file đã nộp và file văn bản gốc bị render lộn vị trí.
- Sửa lỗi frontend nhận response file sai dạng, dẫn tới link PDF bị `undefined` hoặc không mở được.

## 3. Trưởng phòng

- Hiển thị tệp đính kèm văn bản khi trưởng phòng nhận phân công.
- Sửa validate thời hạn phân công:
  - Không cho chọn thời gian quá khứ ở UI.
  - Backend cũng chặn nếu request gửi thời gian quá khứ.
- Cải thiện khu vực phân công nhiệm vụ và hiển thị file liên quan.

## 4. Văn thư

- Bỏ bước xác thực mã 6 số khi ban hành.
- Thêm nút “Đóng dấu & ban hành”.
- Sau khi đóng dấu:
  - Backend tạo bản PDF đã đóng dấu.
  - Lưu file PDF vào `backend-eoffice/uploads/sealed`.
  - Ghi file đã đóng dấu vào bảng file đính kèm của văn bản.
  - Chuyển trạng thái văn bản sang `PUBLISHED`.
- Thêm dòng ngày tháng màu đen trong PDF:
  - `Đà Nẵng, ngày ... tháng ... năm ...`
- Con dấu đỏ được đặt theo kiểu văn bản hành chính:
  - Với PDF hệ thống tự sinh: đặt dưới dòng nội dung cuối.
  - Với PDF upload sẵn: đặt cuối trang cuối vì thư viện hiện tại chưa đọc được chính xác vị trí dòng chữ cuối trong layout PDF có sẵn.

## 5. Backend và bảo mật

- Bật lại kiểm tra role cho nhiều route trước đó đang bỏ qua `checkRole`.
- Giảm rủi ro giả mạo role/userId:
  - Production không tin trực tiếp `x-user-id`, `x-user-role`, query `userId`.
  - Các header này chỉ còn phù hợp cho dev/local.
- Siết upload file:
  - Bỏ `application/octet-stream`.
  - Bỏ `.zip`.
  - Yêu cầu MIME và đuôi file đều hợp lệ.
  - Giới hạn file upload còn 20MB.
  - Cho phép thêm `.md` và `text/markdown`.
- Thêm trạng thái `PUBLISHED` cho document workflow.
- Thêm script patch enum database:
  - `npm run db:patch-workflow-enums`

## 6. Cloudinary PDF

- Phát hiện Cloudinary có thể chặn public delivery PDF/ZIP trên tài khoản free/untrusted.
- Vì vậy file PDF đóng dấu hiện ưu tiên mở qua backend local:
  - `http://localhost:3001/uploads/sealed/...pdf`
- Cloudinary vẫn có thể upload dự phòng, nhưng app không còn phụ thuộc vào link `res.cloudinary.com/.../raw/upload/...pdf` để mở PDF.

## 7. Thư viện mới cần lưu ý

Backend đã thêm thư viện:

- `pdf-lib`
- `@pdf-lib/fontkit`

Nếu người khác pull code về và **chưa có `node_modules`**, cần chạy:

```bash
cd backend-eoffice
npm install
```

Nếu đã pull cả `package-lock.json` và chạy `npm install` bình thường thì không cần cài riêng từng package.

Frontend không cần thêm thư viện mới trong phiên này; chỉ cần chạy `npm install` nếu máy chưa có `node_modules`.

## 8. Lệnh cần chạy sau khi pull

Backend:

```bash
cd backend-eoffice
npm install
npm run db:patch-workflow-enums
npm run dev
```

Frontend:

```bash
cd eoffice-teams-fe
npm install
npm run dev
```

Nếu database đã được patch `PUBLISHED` rồi thì không cần chạy lại `npm run db:patch-workflow-enums`.

## 9. Kiểm tra đã chạy

- Backend load app thành công.
- Frontend build thành công bằng:

```bash
cd eoffice-teams-fe
npm run build
```

## 10. Gợi ý commit chuyên nghiệp

```bash
git status
git add backend-eoffice eoffice-teams-fe CHANGES_2026-05-28.md
git commit -m "feat: improve task file workflow and PDF sealing"
git push origin main
```

Nếu branch hiện tại không phải `main`, kiểm tra bằng:

```bash

```

Rồi push theo branch đó:

```bash
git push origin <ten-branch>
```
